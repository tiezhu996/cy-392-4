## 项目主要需求

### 项目概述
**项目类型：Web前端**
搭建一个手工作品瀑布流展示墙，手工爱好者可上传作品图片并标注材料和教程链接，浏览者以瀑布流形式浏览精美手工作品。

### 功能需求

1. **作品上传**：上传作品图片（支持多张），填写作品名称、手工类型（编织/木工/陶艺/皮具/纸艺/刺绣/其他）、使用材料、制作时长、难度等级（入门/进阶/高级）。
2. **瀑布流展示**：首页以 Pinterest 风格瀑布流展示所有作品，图片自适应高度，无限滚动加载。
3. **作品详情**：点击作品查看图片轮播、详细描述、材料清单、制作步骤（可选）、作者信息。
4. **分类筛选**：按手工类型、难度等级、最新/最热排序筛选作品。
5. **点赞与收藏**：对作品点赞和收藏，收藏夹页面集中管理已收藏作品。
6. **作者主页**：查看作者发布的所有作品列表，关注/取关作者。
7. **搜索功能**：关键词搜索作品名称和描述。
8. **深色模式**：支持亮色/暗色主题切换。

### 技术要求

- 前端框架：Vue 3 + TypeScript
- UI 库：Naive UI + 自定义瀑布流组件（CSS Grid / Masonry 布局）
- 图片懒加载：Intersection Observer API
- 状态管理：Pinia
- 构建工具：Vite
- 数据持久化：浏览器 IndexedDB
- 代码规范：严格遵循单一职责原则，每个文件只负责一个功能；常量（手工类型枚举、难度等级）统一维护在 `src/constants/` 目录下；类型定义统一在 `src/types/`；所有工具函数封装在 `src/utils/`。

## 代码存放规则

这是纯前端单页应用（SPA）项目，无后端、无数据库。图片使用 mock 数据和远程占位图（如 picsum.photos）。

前端代码统一放在 `frontend/`：

```
frontend/
├── src/
│   ├── views/              # 页面视图
│   ├── components/         # 通用组件（瀑布流、作品卡片等）
│   ├── composables/        # 组合式函数
│   ├── stores/             # Pinia 状态管理
│   ├── storage/            # IndexedDB 封装
│   ├── constants/          # 常量定义
│   ├── types/              # TypeScript 类型
│   ├── utils/              # 工具函数
│   ├── mock/               # Mock 数据
│   ├── styles/             # 全局样式
│   └── App.vue
├── public/
└── package.json
```
